# 食物语（FoodGPT）项目需求文档

## 1. 项目概述

### 1.1 项目背景
**食物语（FoodGPT）** 

### 1.2 项目目标

1. 提供基于身体数据的个性化饮食建议
2. 通过数据追踪帮助用户了解自身营养状况
3. 利用 AI 技术生成智能食谱推荐
4. 针对女性用户提供专属周期管理和饮食指导
5. 构建"记录→分析→推荐→干预"的完整健康管理闭环

### 1.3 用户人群

- **核心用户**：关注健康饮食的女性用户（大学生、职场女性为主）
- **用户需求**：减脂、增肌或健康维持
- **年龄范围**：18-35岁

### 1.4 产品形态
- **应用类型**：桌面小程序
- **界面风格**：简洁、直观
- **功能模块**：用户管理、身体数据管理、营养素追踪、智能食谱推荐、周期管理、智能干预提醒等
---

## 2. 功能需求

项目分为两大层次：**基础功能层（全民通用）** 和 **女性友好专区（增强层）**。

### 2.1 基础功能层

#### 2.1.1 用户管理与身体数据管理（B-02）

**功能描述**：
- 用户可录入身高、体重、年龄、活动量（久坐/轻度/中度/活跃）
- 系统自动计算 BMI（身体质量指数）和 BMR（基础代谢率）
- 支持修改和更新身体数据

**用户场景**：
- 用户首次使用应用时，需要录入个人身体数据
- 用户体重发生变化时，需要更新身体数据以获取准确的营养建议

**验收标准**：
- 身高、体重、年龄输入框校验有效范围（身高 100-250cm，体重 30-200kg，年龄 10-100岁）
- 活动量选择项包含：久坐、轻度、中度、活跃
- BMI 计算公式正确：体重(kg) / 身高(m)²
- BMR 计算公式正确（采用 Mifflin-St Jeor 公式）
- 数据修改后实时更新 BMI 和 BMR 显示

#### 2.1.2 体重追踪（B-03）

**功能描述**：
- 用户可定期记录体重
- 系统以折线图展示体重变化趋势

**用户场景**：
- 用户每周固定时间记录体重，观察变化趋势
- 用户在减脂/增肌过程中，需要查看体重变化曲线

**验收标准**：
- 支持手动录入日期和体重
- 折线图展示至少最近 30 天的体重数据
- 图表支持缩放和拖拽查看
- 体重记录支持删除和修改

#### 2.1.3 菜谱管理（B-04）

**功能描述**：
- 用户可手动录入菜谱（名称/分类/食材/步骤/营养素：蛋白质/碳水/脂肪/热量）
- 支持菜谱的增删改查
- 自动记录联网搜索时用户选择保存的菜谱

**用户场景**：
- 用户想记录自己喜欢的家常菜谱
- 用户从网络搜索到心仪的菜谱，一键保存到本地
- 用户需要修改已有菜谱的食材或步骤

**验收标准**：
- 菜谱表单包含：名称、分类（早餐/午餐/晚餐/加餐/零食）、食材列表、烹饪步骤、营养成分（蛋白质/碳水/脂肪/热量）
- 支持按名称、分类搜索菜谱
- 菜谱列表支持分页展示
- 联网搜索的菜谱可自动保存到本地数据库

#### 2.1.4 联网搜索菜谱（B-05）

**功能描述**：
- 用户输入已有食材，调用搜索 API 联网推荐菜谱
- 展示候选菜谱列表，用户选择后自动保存到本地数据库

**用户场景**：
- 用户冰箱里有剩余食材，想知道能做什么菜
- 用户想尝试新菜品，通过关键词搜索菜谱

**验收标准**：
- 搜索框支持输入食材关键词
- 搜索结果展示菜谱名称、图片、简介、热量
- 用户点击"保存"后，菜谱自动存入本地数据库
- 网络异常时显示友好的错误提示

#### 2.1.5 联网搜索增强（B-06）

**功能描述**：
- 根据用户偏好（低脂/高蛋白/素食/无糖等）自动优化搜索关键词

**用户场景**：
- 用户偏好低脂饮食，搜索时自动追加"低脂"关键词
- 用户是素食者，搜索结果自动过滤含肉类的菜谱

**验收标准**：
- 用户可设置饮食偏好标签（低脂、高蛋白、素食、无糖、低碳水）
- 搜索时自动将偏好标签追加到搜索请求中
- 搜索结果优先展示符合用户偏好的菜谱

#### 2.1.6 用餐记录（B-08）

**功能描述**：
- 用户记录每日饮食（菜品、份量、餐次：早餐/午餐/晚餐/加餐）
- 系统自动记录最近 N 天的饮食历史

**用户场景**：
- 用户餐后记录吃了什么，便于后续营养分析
- 用户需要查看昨天或上周的饮食记录

**验收标准**：
- 支持选择餐次（早餐、午餐、晚餐、加餐）
- 支持从本地菜谱库中选择菜品或手动输入
- 份量输入支持常见单位（份、克、毫升）
- 展示最近 7 天的饮食记录列表
- 支持删除和修改用餐记录

#### 2.1.7 营养素分析（B-09）

**功能描述**：
- 根据用餐记录自动计算每餐及全天的蛋白质/碳水/脂肪摄入量
- 按日期汇总展示

**用户场景**：
- 用户想知道今天摄入了多少蛋白质和碳水
- 用户需要查看某一天的营养素摄入明细

**验收标准**：
- 自动计算每餐的蛋白质、碳水、脂肪摄入量
- 汇总计算全天的营养素摄入总量
- 按日期展示历史营养素摄入数据
- 数据精度保留一位小数

#### 2.1.8 营养素进度（B-10）

**功能描述**：
- 以进度条形式展示当日三大营养素摄入情况（已摄入/目标值）

**用户场景**：
- 用户在主页查看今日营养摄入进度
- 用户想知道还需要摄入多少蛋白质才能达到目标

**验收标准**：
- 展示蛋白质、碳水、脂肪三个进度条
- 每个进度条显示已摄入量/目标值
- 进度条颜色根据完成度变化（未达标：橙色，达标：绿色）
- 实时更新进度数据

#### 2.1.9 均衡度判断（B-11）

**功能描述**：
- 分析当日营养结构是否均衡，生成 0-100 的综合评分

**用户场景**：
- 用户想了解今日饮食是否均衡
- 用户根据评分调整后续饮食计划

**验收标准**：
- 根据三大营养素的摄入比例计算均衡度评分
- 评分范围 0-100，60 分以下为不均衡，80 分以上为优秀
- 显示评分等级和简要评语
- 点击查看详细分析报告

#### 2.1.10 主动干预（B-12）

**功能描述**：
- 每天自动生成营养报告
- 针对营养缺口给出具体补充建议（如"蛋白质不足，建议晚餐补充鸡胸肉"）

**用户场景**：
- 用户打开应用时收到当日营养报告
- 用户收到针对性的营养补充建议

**验收标准**：
- 每日首次打开应用时自动显示营养报告
- 报告包含当日摄入总结、均衡度评分、营养缺口分析
- 根据缺口生成具体的食材建议
- 建议内容具体可操作，包含食材名称和推荐摄入量

### 2.2 女性友好专区（增强层）

专区复用基础功能的身体数据、用餐记录和营养素分析结果，在此基础上增加以下功能：

#### 2.2.1 健康目标设定（F-01）

**功能描述**：
- 用户可选择目标：减脂 / 增肌/增重 / 维持健康

**用户场景**：
- 用户开始使用时设定健康目标
- 用户根据自身情况调整目标

**验收标准**：
- 目标选项包含：减脂、增肌/增重、维持健康
- 选择目标后系统自动调整推荐热量范围
- 支持随时修改目标

#### 2.2.2 生理周期管理（F-02）

**功能描述**：
- 用户记录月经周期起止日期
- 系统自动识别当前周期阶段，动态调整推荐策略（如经期推荐补铁食谱）

**用户场景**：
- 用户记录本次月经开始日期
- 用户在经期收到特殊饮食建议

**验收标准**：
- 支持录入月经周期开始日期和周期长度
- 系统自动计算下次月经预测日期
- 识别当前周期阶段（经期、卵泡期、排卵期、黄体期）
- 不同周期阶段推荐不同的饮食策略

#### 2.2.3 女生专属食谱库（F-03）

**功能描述**：
- 内置五类食谱：减脂、增肌、健康维持、经期调理、孕期调理

**用户场景**：
- 用户在经期想查看适合的调理食谱
- 用户准备怀孕或怀孕期间需要特殊饮食指导

**验收标准**：
- 食谱库包含五类：减脂、增肌、健康维持、经期调理、孕期调理
- 每类至少包含 20 道精选食谱
- 食谱标注适用周期阶段
- 支持按类别筛选和搜索

#### 2.2.4 热量目标计算（F-04）

**功能描述**：
- 基于 BMR × 活动系数 + 目标调整值，自动计算每日推荐热量范围

**用户场景**：
- 用户设定目标后，系统自动计算每日热量需求
- 用户活动量变化时，热量目标自动更新

**验收标准**：
- 根据 BMR × 活动系数计算基础热量需求
- 根据目标类型调整热量（减脂：-300~500kcal，增肌：+300~500kcal）
- 显示每日推荐热量范围
- 身体数据或活动量变化时自动重新计算

#### 2.2.5 食物偏好录入（F-05）

**功能描述**：
- 用户可设置偏爱食材和忌口食材

**用户场景**：
- 用户不喜欢吃香菜，设置为忌口
- 用户喜欢吃海鲜，设置为偏爱

**验收标准**：
- 支持添加偏爱食材和忌口食材列表
- 偏爱食材在推荐时优先考虑
- 忌口食材在推荐时自动过滤

#### 2.2.6 专属食谱匹配（F-06）

**功能描述**：
- 从菜谱库中过滤出符合热量目标 + 偏好 + 忌口的菜谱

**用户场景**：
- 用户想找到既符合减脂目标又不含忌口食材的菜谱
- 用户根据当前周期阶段获取专属推荐

**验收标准**：
- 过滤条件包含：热量范围、偏爱食材、忌口食材、周期阶段
- 推荐结果按匹配度排序
- 显示匹配度评分
- 支持保存匹配结果到收藏

#### 2.2.7 搜索关键词增强（F-07）

**功能描述**：
- 搜索时自动追加女性专属关键词 + 用户偏好

**用户场景**：
- 用户搜索菜谱时，结果自动偏向女性友好的选项
- 用户在经期搜索时自动追加"经期调理"关键词

**验收标准**：
- 根据当前周期阶段自动追加相关关键词
- 根据用户偏好追加偏好关键词
- 搜索结果优先展示符合条件的菜谱

#### 2.2.8 营养缺口分析（F-08）

**功能描述**：
- 对比用户目标与当日实际摄入，识别缺少的营养素

**用户场景**：
- 用户想知道今天哪些营养素摄入不足
- 用户根据缺口调整明天的饮食计划

**验收标准**：
- 对比每日目标摄入量与实际摄入量
- 识别缺口超过 20% 的营养素
- 展示缺口比例和具体数值
- 按缺口严重程度排序

#### 2.2.9 行动建议生成（F-09）

**功能描述**：
- 根据营养缺口生成具体的食材建议（如"缺铁，建议吃菠菜/红肉"）

**用户场景**：
- 用户收到针对性的营养补充建议
- 用户根据建议调整餐食选择

**验收标准**：
- 针对每种缺口营养素生成至少 3 种食材建议
- 建议考虑用户的饮食偏好和忌口
- 建议食材常见易获取
- 显示每种食材的营养成分含量

#### 2.2.10 AI 健康顾问（F-10）

**功能描述**：
- 结合用户身体数据 + 周期阶段 + 目标 + 偏好 + 历史饮食记录
- 调用大模型（DeepSeek）生成个性化饮食建议

**用户场景**：
- 用户想获得更全面的个性化饮食指导
- 用户有特殊健康问题需要咨询

**验收标准**：
- 整合用户的身体数据、周期阶段、目标、偏好和历史饮食
- 调用 DeepSeek 大模型生成自然语言建议
- 建议内容具体、实用、个性化
- 支持对话式交互，用户可进一步提问
- 联网异常时显示离线模式提示

## 3. 非功能需求

### 3.1 应用类型
- 桌面客户端，双击即可运行

### 3.2 数据存储
- 本地数据库（MyBatis-Plus）
- 支持离线查看已保存的菜谱和记录

### 3.3 联网功能
- 搜索菜谱和 AI 咨询时需要联网
- 其他功能离线可用

### 3.4 响应速度
- 界面操作流畅，无明显卡顿
- 进度条实时更新
- 数据加载时间不超过 2 秒

### 3.5 跨平台
- 支持 Windows/macOS（JavaFX 天然跨平台）
- 界面在不同分辨率下自适应

### 3.6 数据安全
- 用户数据仅存储在本地
- 支持数据备份和恢复功能(简单方式：仅复制数据库文件)

### 3.7 可用性
- 界面操作直观，无需培训即可使用
- 提供清晰的错误提示和操作引导
- 支持常用快捷键（如 Ctrl+S 保存、Ctrl+F 搜索）

## 5. 约束与假设

### 5.1 技术约束
- 采用 JavaFX 框架开发，确保跨平台兼容性
- 使用 MyBatis-Plus 数据库存储数据
- 联网功能依赖第三方菜谱搜索 API
- AI 咨询功能依赖 DeepSeek 大模型 API

### 5.2 数据约束
- 用户身体数据需符合合理范围（身高 100-250cm，体重 30-200kg）
- 菜谱营养成分数据需准确录入
- 生理周期数据由用户手动录入，系统不进行医疗判断
- 联网搜索结果质量依赖第三方 API 返回数据

### 5.3 功能约束
- AI 建议仅供参考，不能替代专业医疗建议
- 孕期调理食谱仅供参考，用户应咨询医生意见
- 网络异常时，搜索和 AI 功能不可用，但不影响本地数据操作

### 5.4 假设条件
- 用户具备基本的计算机操作能力
- 用户拥有稳定的网络连接（用于搜索和 AI 功能）
- 用户愿意定期录入身体数据和饮食记录
- 用户了解应用提供的是健康建议而非医疗诊断
